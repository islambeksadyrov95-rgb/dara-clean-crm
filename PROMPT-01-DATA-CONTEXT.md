## Источники данных

### 1. Финансы (ДДС)

**Файл:** `D:/Mind map/Dara Clean/Консультация/Финансы/Финансы DaraClean 2025-2026.xlsx`

**Листы:**
- `Справочник` — категории (dropdown values)
- `Ежедневно` — 3,587 транзакций за 2025–2026 (дата, тип, оплата, сумма, категория, статья, сотрудник, комментарий)
- `ДДС 2025` — сводный ДДС по месяцам (строки 1–119)
- `ДДС 2026` — сводный ДДС по месяцам (та же структура, заполнен Q1)
- `ЗП ежедневн` — ежедневный учёт зарплат
- `Октябрь` — детализация по дням октября

### 2. Продажи

**Файл:** `D:/Mind map/Dara Clean/Консультация/Продажи/Отдел Продаж Апрель.xlsx`

**Структура:** 33 листа — `Ввод`, `Справочники`, `Аналитика`, + 30 ежедневных листов (01.04–30.04)

**Колонки ежедневных листов (11):**

| Колонка | Описание |
|---------|----------|
| Время | Время обращения |
| Менеджер | Имя менеджера (Самал, Елена, Рауза) |
| Телефон клиента | Номер |
| Канал | WhatsApp номер / Звонок |
| Источник | Google Реклама, 2GIS, Instagram, TikTok, Рекомендация, Постоянный клиент, Старая база |
| Тип клиента | Новый / Повторный / Постоянный / Старая база |
| Услуга | Ковры, Шторы/Тюль, Химчистка мягкой мебели и т.д. |
| Статус | Заказ оформлен / Думает / Отказ / Не отвечает |
| Причина отказа | Дорого / Не устроила минималка / Просто спросил и пропал |
| Перезвон | Да / Нет |
| Комментарий | Свободный текст |

**ВАЖНО:** В таблице продаж НЕТ колонки суммы/чека. Это трекинг обращений, не заказов. Суммы берутся из ДДС.

## ETL Pipeline

**Существующий:** `D:/Mind map/Dara Clean/etl/merge_build.py` — собирает все источники → `dashboard-data.json`

**Для нового дашборда нужен дополнительный JSON:**

Файл: `data/finance-full-data.json`

```json
{
  "costModel": { ... },       // Себестоимость по блокам (из PROMPT-02)
  "financialPlan": { ... },   // Планирование и кассовый разрыв (из PROMPT-03)
  "salesPlan": { ... },       // Планы продаж (из PROMPT-04)
  "unitEconomics": { ... }    // CAC, LTV, unit-экономика (из PROMPT-02)
}
```

---

## Связь между промтами

```
PROMPT-01 (этот файл) — контекст данных, используется ВСЕМИ
    ↓
PROMPT-02 — модель себестоимости + unit-экономика (CAC, LTV, кв.м.)
    ↓
PROMPT-03 — финансовый план, кассовый разрыв, сценарии
    ↓
PROMPT-04 — таблицы продаж, воронка, аналитика по менеджерам
    ↓
PROMPT-05 — UI/UX дашборда, компоненты, интерактивность
```
