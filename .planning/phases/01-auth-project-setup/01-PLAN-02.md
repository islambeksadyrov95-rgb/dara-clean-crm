# Plan: Middleware route protection + role-based access + stub pages

<objective>
Wire up Next.js middleware so every protected route validates the session with Supabase getUser(), unauthenticated visitors are redirected to /login, and the /import route is restricted to admin role. Create stub pages for /queue and /import so the full auth flow is testable end-to-end. After this plan, the three phase success criteria are met: manager lands on /queue, admin can access /import, logged-out user is redirected to /login.
</objective>

<must_haves>
## Truths (from CONTEXT.md decisions)
- middleware.ts uses getUser() — never getSession()
- Unauthorized route: redirect to role's default page (not to /login — avoids redirect loops for authenticated users)
- /import is admin-only; manager hitting /import redirects to /queue
- Both manager and admin land on /queue after login
- Route group (protected) wraps all auth-required pages

## Must-Have Outcomes
- Navigating to /queue or /import without a session redirects to /login
- manager@test.com can access /queue but NOT /import (gets redirected to /queue)
- admin@test.com can access both /queue and /import
- Logged-in user navigating to /login is redirected to /queue
- `npm run build` passes after these changes
</must_haves>

Create crm/middleware.ts. This file runs on every request and enforces auth + role rules.

```typescript
import { createServerClient } from '@supabase/ssr'
import { NextResponse, type NextRequest } from 'next/server'

// Маршруты, доступные только для админа
const ADMIN_ROUTES = ['/import']

// Публичные маршруты (не требуют авторизации)
const PUBLIC_ROUTES = ['/login']

export async function middleware(request: NextRequest) {
  let supabaseResponse = NextResponse.next({
    request,
  })

  const supabase = createServerClient(
    process.env.NEXT_PUBLIC_SUPABASE_URL!,
    process.env.NEXT_PUBLIC_SUPABASE_ANON_KEY!,
    {
      cookies: {
        getAll() {
          return request.cookies.getAll()
        },
        setAll(cookiesToSet) {
          cookiesToSet.forEach(({ name, value }) =>
            request.cookies.set(name, value)
          )
          supabaseResponse = NextResponse.next({
            request,
          })
          cookiesToSet.forEach(({ name, value, options }) =>
            supabaseResponse.cookies.set(name, value, options)
          )
        },
      },
    }
  )

  // ВАЖНО: использовать getUser() а не getSession() — проверяет с сервером Supabase
  const {
    data: { user },
  } = await supabase.auth.getUser()

  const { pathname } = request.nextUrl

  // Публичный маршрут — пропускаем
  const isPublic = PUBLIC_ROUTES.some((route) => pathname.startsWith(route))

  if (isPublic) {
    // Если пользователь уже авторизован и идёт на /login — перенаправить на /queue
    if (user) {
      const url = request.nextUrl.clone()
      url.pathname = '/queue'
      return NextResponse.redirect(url)
    }
    return supabaseResponse
  }

  // Не авторизован — на логин
  if (!user) {
    const url = request.nextUrl.clone()
    url.pathname = '/login'
    return NextResponse.redirect(url)
  }

  // Проверка admin-only маршрутов
  const isAdminRoute = ADMIN_ROUTES.some((route) => pathname.startsWith(route))
  if (isAdminRoute) {
    const role = user.user_metadata?.role as string | undefined
    if (role !== 'admin') {
      // Менеджер пытается попасть на admin-only страницу — на /queue
      const url = request.nextUrl.clone()
      url.pathname = '/queue'
      return NextResponse.redirect(url)
    }
  }

  return supabaseResponse
}

export const config = {
  matcher: [
    /*
     * Применяем ко всем маршрутам кроме:
     * - _next/static (статика)
     * - _next/image (оптимизация изображений)
     * - favicon.ico
     * - публичные файлы в /public
     */
    '/((?!_next/static|_next/image|favicon.ico|.*\\.(?:svg|png|jpg|jpeg|gif|webp)$).*)',
  ],
}
```

import { createClient } from '@/lib/supabase/server'
import { redirect } from 'next/navigation'

export const dynamic = 'force-dynamic'

export default async function ProtectedLayout({
  children,
}: {
  children: React.ReactNode
}) {
  const supabase = await createClient()

    data: { user },

  // Middleware должен был поймать это раньше, но на всякий случай

    redirect('/login')

  const role = user.user_metadata?.role as string | undefined

  return (
    <div className="min-h-screen bg-background">
      {/* Временная шапка — будет заменена в Phase 4 */}

          {email} · {role ?? 'неизвестная роль'}

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser → Next.js middleware | Session cookie crosses here on every request |
| Next.js → Supabase Auth | getUser() validates token with Supabase server |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Spoofing | middleware.ts | mitigate | Use getUser() not getSession() — getUser() re-validates with Supabase server, getSession() trusts local cookie |
| T-01-02 | Elevation of Privilege | /import route | mitigate | Role check in middleware: user_metadata.role !== 'admin' → redirect to /queue |
| T-01-03 | Information Disclosure | .env.local | accept | NEXT_PUBLIC_ keys are anon-only (no service role key yet). RLS (Phase 2) prevents data access even with anon key |
| T-01-04 | Tampering | user_metadata.role | accept | Roles set by admin in Supabase dashboard only. No self-registration. Phase 9 adds service role key for Phase 3 admin actions |
</threat_model>
