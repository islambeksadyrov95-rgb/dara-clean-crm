- **WhatsApp достижим 3 раза** на одном экране (`call-work-panel.tsx`: «Написать» :496, «4 Отправить WhatsApp» :590, «Отправить WhatsApp» :597). Две дисозиции-WhatsApp визуально идентичны.
- **Дисозиция = плоская стена из ~8 одинаковых outline-кнопок** (+4 в «Пропустить») на КАЖДЫЙ звонок (`call-work-panel.tsx:582-614`).
- **Дисозиция уезжает под фолд** (история звонков + VPBX-записи раскрыты по умолчанию `:187-188`) — на каждом звонке скролл-охота между «Позвонить» (верх) и логированием (низ).

- **Выбор строки 2 способами**: чекбокс + кнопка «Выбрать».
- **Дисозиция расходится между /queue (плоский список) и /clients (drill-down) + разные подписи** одного действия.
→ **План:** 2-уровневый cockpit (3 цветных primary: Заказ/WhatsApp/Перезвон + overflow «Другое»), sticky-дисозиция под кнопкой звонка, история/записи свёрнуты во время звонка, глобальные хоткеи (звонок/WhatsApp/следующий), единые подписи через `STATUS_LABELS`. Все server actions сохраняются — меняется только поверхность.

### 3. /queue быстрее + bulk
**Почему быстрее:** только `/queue` — Server Component с `Promise.all` из 8 `prefetchQuery` + `HydrationBoundary` + `getClaims()` (локально) + auth-через-контекст. Данные приходят в HTML, на клиенте нет водопада экшенов. Остальные страницы — `'use client'` + клиентский `getUser()` + серилизованные Server Actions.
**Нюанс (живой замер):** на ХОЛОДНОЙ загрузке `/queue` (LCP 2033мс) ≈ `/clients` (1995мс) — обе упираются в латентность Сиднея + гидрацию тяжёлых компонентов; у `/queue` ещё CLS 0.28. Преимущество queue реально только в SPA-навигации. → лечить ТРИ рычага: (а) SSR-prefetch на все страницы, (б) регион БД/функций, (в) вес клиента + CLS.
**Bulk (правда):** чекбоксы на `/queue` есть, и для **admin** есть 2 массовые операции (назначить менеджера/сегмент) через кривой raw `<select>` fire-on-change без Apply/подтверждения (`queue-client.tsx:578-630`, дубль в `clients/page.tsx:397-461`). Для **менеджера эти операции скрыты → он видит чекбоксы, которые ничего не делают.** Нет выбора колонок/видов.
→ **План:** единый `BulkActionBar` (shadcn Select + явный Apply + confirm на «всю выборку») для /queue и /clients; операции: назначить менеджера, сегмент, тег, в рассылку, snooze, экспорт xlsx; решить scope для менеджеров; серверная перепроверка владения по id (анти-IDOR на bulk).
