# Dara Clean CRM

## What This Is

CRM-система повторных продаж для химчистки ковров Dara Clean (Алматы). Менеджеры обзванивают клиентов по базе, создают заказы, отправляют WhatsApp напоминания. Руководитель видит аналитику, настраивает мотивацию и оценивает финансовый эффект скидок.

## Core Value

Менеджер открывает систему, видит кому звонить сегодня, звонит, и за 2 клика создаёт заказ или отправляет WhatsApp — без Excel, без ручного поиска, без дублирования звонков между менеджерами.

## Requirements

### Validated

- Data: 21 388 записей клиентов (июль 2025 — апрель 2026) — existing in База Агбис.xlsx
- Dashboard: финансовая аналитика (ДДС, себестоимость, воронка) — existing in dashboard/
- Telegram Bot: ввод финансовых операций — existing in Telegram Bot/

### Active

- [ ] Импорт клиентской базы из Excel в Supabase с дедупликацией по телефону
- [ ] Аналитика клиентов: сегментация (новый/повторный/постоянный), repeat rate, средний чек
- [ ] CRM обзвона: список клиентов за N дней назад, кнопки дозвонился/нет
- [ ] Создание заказа из CRM: 4 услуги (ковры/шторы/мебель/клининг), авто-скидка
- [ ] WhatsApp сообщения: генерация через OpenRouter, deep link на wa.me
- [ ] Система мотивации: KPI калькуляторы для менеджеров, курьеров, цеха
- [ ] Финансовый калькулятор: эффект скидок на план 2026 с учётом кассового разрыва
- [ ] Сетка скидок: 5% / 10% (>30К) / 15% (комплекс), срок 7 дней
- [ ] Многопользовательский режим: 3-5 менеджеров + руководитель
- [ ] Деплой на Vercel + Supabase

### Out of Scope

- GPS трекинг курьеров — нет инфраструктуры, отложено
- Интеграция с Агбис API — пока ручной импорт Excel, API потом
- Автоматическая отправка WhatsApp — только генерация + ссылка
- Мобильная версия — менеджеры работают на компе
- Авторизация OAuth — простой логин по email/паролю через Supabase Auth

## Context

**Бизнес**: химчистка ковров, штор, мебели, клининг в Алматы. 5 филиалов. 70+ заказов/день в пиковые месяцы.

**Проблема**: нет системы повторных продаж. Клиенты заказывают 1 раз и теряются. Конкурент (Dara Clean — другая компания с тем же названием) системно работает со скидками и CRM, мы — нет.

**Цель консалтинга**: увеличить долю повторных клиентов с текущих ~X% до 30-40%. Каждый повторный клиент = 0 затрат на маркетинг (vs 500-1000 тг CAC на нового).

**Финансовый контекст**:
- Выручка 2025: 113.2М тг, маржа 15.3%
- Факт Q1 2026: 18.7М (рост ~21% год к году)
- Кассовый разрыв: -3.26М с 2025 года
- Кредит: 364 605 тг/мес с апреля 2026
- Средний чек: 17-18К тг, ~1 050 тг/м2
- Переменная доля расходов: 30% (blended)

**Мотивация сотрудников**: нужна прозрачная система где менеджеры и курьеры зарабатывают больше от результата. Компания должна зарабатывать больше плана, а потом делиться — не наоборот.

**Данные клиентов**: База Агбис.xlsx — 21 388 строк. Колонки: дата заказа, имя, адрес, телефон, стоимость, услуга. Группировка по датам. openpyxl падает из-за невалидных цветов в XML — нужен monkey-patch.

## Constraints

- **Timeline**: MVP в боевом режиме к 15 мая 2026 (завтра)
- **Tech stack**: Next.js + Supabase (PostgreSQL + Auth) + Vercel
- **Users**: 3-5 менеджеров + руководитель, все через браузер на компе
- **Data source**: Excel импорт (База Агбис.xlsx), позже API Агбис
- **AI**: OpenRouter API для генерации WhatsApp сообщений
- **Budget**: Supabase Free/Pro, Vercel Free/Pro, OpenRouter pay-per-use

## Key Decisions

| Decision | Rationale | Outcome |
|----------|-----------|---------|
| Next.js + Supabase вместо vanilla JS | 3-5 менеджеров одновременно, нужна общая БД и auth | — Pending |

| OpenRouter для WhatsApp вместо шаблонов | Пользователь хочет AI-генерацию, ~$0.001/сообщение | — Pending |
