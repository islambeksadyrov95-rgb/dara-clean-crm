# Design Spec: Очередь звонков (/queue) — редизайн

Date: 2026-06-02
Selected approach: **A — «Командный центр 2.0»** (выбран в visual companion, event `A-cockpit`)

## Цель
Объединить 3 направления, НЕ меняя бизнес-логику queue/page.tsx:
- A — карточка звонка как герой с закреплённой зоной действий внизу
- B — единая палитра + иерархия карточек (тени/веса), меньше микро-лейблов
- C — приоритетная очередь + компактный «План дня» одной строкой

## Layout
Двухколоночный grid `1.55fr / 1fr` (как сейчас flex, оставить). Без активного клиента — список на всю ширину.

### Верх: «План дня» одной строкой
Заменить текущую карточку (3 прогресс-бара + дублирующая сетка 4 колонок) на одну компактную строку-карточку справа от заголовка:
`Звонки 18/40 [bar] · Заказы 3/5 [bar] · Выручка 51К/85К ₸ · [Конв. 17%]`
Мини-бары шириной ~60px. Цвета: звонки=accent(blue), заказы=ok(green), выручка=sky.

### Лево: приоритетная очередь
- Чипы-пресеты (Все / Повторные / В риске / Потерянные) + чекбокс «позвоненные» — restyle в pill-чипы.
- Таблица колонки: `#` (порядок-приоритет) · Клиент (имя + телефон·адрес мелким) · Сегмент (pill) · Дней (right) · действие.
- Строка №1 (следующий к звонку) подсвечена: `bg #fafcff` + левая полоса `inset 3px 0 accent`, кнопка `Звонить →` (primary). Остальные — кнопка `Звонить` (outline).
- «Перезвоны сегодня» — оставить отдельной карточкой (янтарной) над таблицей, если есть.
- Низ: «Показано X из Y · приоритет по сегменту и дням» + «Загрузить ещё».

### Право: карточка звонка (sticky, hero)
Карточка `dq-card` + мягкая тень, flex-column, overflow hidden:
1. Шапка: имя (18px/700), телефон, pill сегмента справа; строка мета `72 дн · 4 заказа · 68 000 ₸ · попытка 1/3`.
2. Тело (gap 12): скрипт в светлом боксе (`#f8fafc`), затем сворачиваемые «История звонков (N)» и «🎙 Запись и транскрипт» (collapsible — по умолчанию свёрнуты).
3. AI-оценка (ScoreDisplay) — после обработки, как секция.
4. **Закреплённая зона действий внизу** (`margin-top:auto`, верхняя граница, фон `#fcfcfd`): лейбл «Результат звонка» + 2 кнопки `Дозвонился`(green) / `Не дозвонился`(red), подсказка `D / N · Esc`.
   - Подэтапы (reached_actions / not_reached_actions / decline_reason / callback_schedule / order / whatsapp) рендерятся в этой же нижней зоне, в той же визуальной стилистике (кнопки в столбик, «← Назад»). OrderForm/WhatsAppPanel — без изменения логики, только консистентные отступы/кнопки.

## Палитра (семантика, единая)
- База: нейтральная shadcn (--primary near-black, серые). Акцент: blue `#2563eb` (CTA «Звонить», прогресс звонков, лейбл скрипта).
- Сегменты RFM (pill, soft tint):
  - Новый → blue (`bg #eff6ff / text #1d4ed8`)
  - Повторный → teal (`#f0fdfa / #0f766e`)
  - Постоянный → green (`#ecfdf5 / #047857`)
  - В риске → amber (`#fffbeb / #b45309`)
  - Потерянный → red (`#fef2f2 / #b91c1c`)
- Действия: Дозвонился=green `#16a34a`, Не дозвонился=red `#dc2626`.
- Статусы истории: reached=green, declined=red, прочее=muted.

Вынести цвета сегментов в общий объект-константу (заменить текущий `SEGMENT_COLORS`), переиспользовать в таблице, карточке, истории.

## Components (shadcn/ui — уже есть)
Card, Table, Badge, Button, Input, Checkbox, Select, Label. Доп. компоненты не нужны.
Collapsible — нативный `<details>`/`<summary>` или локальный useState-тоггл (без новой зависимости).

## States
- Loading: строка таблицы «Загрузка...» (оставить) → можно skeleton-строки (опц.).
- Empty: «Нет клиентов в очереди» — оформить как мягкий empty-state.
- Error: добавить, если fetchQueue упадёт (сейчас нет) — toast (sonner уже есть).
- Active call: правая панель видна только при выбранном клиенте.

## Responsive
Десктоп-only (менеджеры за ПК — по PROJECT.md). Минимум ширины ~1200px; ниже — список на всю ширину, панель скрыта/скролл. Мобайл вне scope.

## Сохранить (non-regression)
- Вся логика queue/page.tsx: lock/unlock, фазы диспозиции, горячие клавиши D/N/Esc, авто-выбор первого, realtime, refresh 30s, getDayStats/getScheduledCallbacks.
- Поля client_segments (rfm_segment, days_since_last_order, last_called_at и т.д.).
- OrderForm / WhatsAppPanel / CallTranscript / ScoreDisplay — публичные пропсы без изменений.

## Файлы (ожидаемо)
- `app/(protected)/queue/page.tsx` — основная вёрстка (главное).
- Возможно мелкие правки стилей в `order-form.tsx`, `whatsapp-panel.tsx`, `score-display.tsx`, `call-transcript.tsx` для консистентности кнопок/отступов.
- Цвета сегментов — общая константа (в page.tsx или отдельный `lib/segments.ts`).
