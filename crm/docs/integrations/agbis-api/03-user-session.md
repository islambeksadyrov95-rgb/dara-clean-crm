# Agbis API — Команды ТОЛЬКО с пользовательской сессией

> **Источник:** https://doc.minihim.ru/api/user_session — «3. Команды API ТОЛЬКО с пользовательской сессией»
>
> Все команды на этой странице **требуют пользовательскую сессию**: параметр `SessionID` (он же `Session_id`), который выдаёт команда `Login`. Сессия живёт **10 минут**, продлевается командой `RefreshSession`.
>
> **Все строковые значения в ответах URL-encoded** (например `%D0%90%D0%B3%D0%B1%D0%B8%D1%81`, `+`/`%20` = пробел, `%2B` = `+`). Перед использованием декодируй (`decodeURIComponent`).
>
> Базовый адрес запроса: `GET/POST .../api/?...`

## Общие соглашения

- **`error`** — статус выполнения, во всех ответах:
  - `0` — нет ошибки
  - `1` — ошибка
  - `2` — не авторизирован
  - `3` — авторизация просрочена
- **`Msg`** — описание ошибки (присутствует при `error = 1`).
- **`callback_address`** (для тяжёлых POST-команд, c версии 24.4) — https-адрес, на который придёт `POST` с результатом. Если указан, в ответ сразу отдаётся подтверждение постановки задачи в очередь, а результат уходит на адрес. При недоступности адреса система делает **5 попыток** отправки, затем прекращает. Используй его (или уменьшай период), т.к. большие периоды могут падать по timeout.
- Заголовок для POST: `Content-type: application/json; charset=UTF-8` (обязателен), `Accept-Encoding: deflate` (необязателен, сжатие ответа).

## Состав страницы

**Актуальные:**
- Справочная информация: `GetListsOrderTNDForAll`
- Работа с клиентами: `ContrInfoForAll`, `ContragInfoForAll`, `BonusesBetweenForAll`, `ClientsBonusRestForAll`

## Справочные enum'ы (критично)

> Эти коды переиспользуются во множестве полей ниже.

**Статус заказа / услуги (`status`, `status_id`):**
- `1` — новый
- `3` — в исполнении
- `4` — исполненный
- `5` — выданный
- `7` — отменённый

(в `OrderInfoForAll` для услуг перечислено `1 – новый, 3 – в исполнении, 4 – исполненный, 5 – выданный`; поле `status` у заказа — устаревшее.)

**Тип заказа (`kind_id`):**
- `0` — заказ химчистки
- `1` — прачечный заказ
- `2` — продажа товаров
- `4` — выездной заказ

**Тип клиента (`sp`):**
- `1` — организация
- `2` — физическое лицо

**Пол клиента (`gender`):**
- `0` — мужской
- `1` — женский

**Подтверждение от клиента (`waiting_confirm`):**
- `0` — не требует подтверждения (по умолчанию)
- `1` — требует подтверждения от клиента
- `2` — клиент согласился, подтверждён
- `3` — клиент НЕ согласился, отменён

**Подтверждение сотрудником химчистки (`is_not_confirmed`):**
- `0` — сотрудник уже подтвердил заказ либо подтверждение не ожидается (по умолчанию)
- `1` — сотрудник ещё не подтвердил, но подтверждение ожидается

**Статус накладной в пути (`diw_status_id`):**
- `1` — Готова к отгрузке
- `2` — В пути
- `3` — Принята
- `4` — Принята не полностью

**Тип платежа (`doc_type`) в `OrderPaysBetweenForAll`:**
- `3` — Кассовый документ
- `4` — Банковский документ
- `9` — Оплата банковской картой
- `31` — Кассовый чек (оплата покупателем)
- `32` — Кассовый чек (возврат денег покупателю)
- `91` — Бонус
- `92` — Оплата депозитом

## GetListsOrderTNDForAll

Получение всех списков срочности, видов детской одежды и наценки. Метод: **GET**, header не требуется.

```
GET .../api/?GetListsOrderTNDForAll&SessionID=...
```

Параметры:
- `SessionID` — сессия (**обязательный**).

```json
{
  "error": 0,
  "order_dirties":   [ { "id": "1001", "name": "%D0%9E%D1%87%D0%B5%D0%BD%D1%8C..." } ],
  "order_nurseries": [ { "id": "1001", "name": "%D0%94%D0%B5%D1%82%D1%81%D0%BA%D0%B0%D1%8F" } ],
  "order_times":     [ { "id": "1002", "name": "%D0%9D%D0%B5%20%D1%81%D1%80%D0%BE%D1%87%D0%BD%D1%8B%D0%B9" } ]
}
```

Поля:
- `error` — статус (см. enum'ы).
- `order_dirties` — массив строк **наценки** услуг (`id`, `name`).
- `order_nurseries` — массив строк **видов детской одежды** для услуг (`id`, `name`).
- `order_times` — массив строк **типов срочности** заказа (`id`, `name`).
- `Msg` — описание ошибки (при `error = 1`).

- `address` — адрес клиента.

- `agree_to_receive_adv_sms` — согласен получать рекламные и поздравительные СМС.
- `barcode` — номер карточки клиента.
- `discount` — процент скидки клиента.
- `scheme_id` — ID дисконтной схемы.
- `discount_scheme_name` — наименование дисконтной схемы.
- `is_family` — признак семейная скидка.
- `gender` — пол клиента (`0` — мужской, `1` — женский).
- `sp` — тип клиента (`1` — организация, `2` — физическое лицо).
- `source` — любой текст.
- `date_create` — дата создания.
- `agree_save_card_pay` — согласие клиента на сохранение карты оплаты.
- `birth_day` — дата рождения.

- `price_list_id` — ID прайс-листа по умолчанию (с версии 24.4).
- `allow_order_out_without_pay` — разрешать ли выдачу заказа без оплаты.
- `order_not_pay` — количество не оплаченных заказов.
- `order_count` — количество заказов.
- `order_completed` — количество заказов на хранении и выданных.
- `deposit` — остаток депозита клиента.
- `bonus` — остаток бонуса клиента.
- `dolg` — сумма долга клиента.
- `promo_code` — используемый промо-код.
- `lng` — язык (в примере `"RU"`).
- `need_confirm_contract` — нужно ли подтверждение договора.
- `version_contract` — версия договора.
- `addresses` — массив адресов.
- `date_last_ds` — дата присвоения последней ДС.

## ContragInfoForAll

Получение информации о клиенте **по телефону**. Метод: **GET**, header не требуется.

```
ContragInfoForAll={"teleph_cell":"+79990009900"}&SessionID=
GET .../api/?ContragInfoForAll=%7B%22teleph_cell%22%3A%22%2B79990009900%22%7D&SessionID=...
```

Параметры:
- `teleph_cell` — номер телефона (**обязательный**). В начале должен быть `+`, кодируется как `%2B`.
- `SessionID` — ID сессии.

Ответ — поля:
- `error` — статус.
- `contr_id` — ID клиента.
- `name` — короткое имя.
- `fullname` — полное имя.
- `teleph_cell` — сотовый телефон (в начале `+` → `%2B`).
- `email` — электронная почта.
- `folder_id` — ID группы клиента.
- `address` — адрес.
- `barcode` — номер дисконтной карты.
- `discount` — скидка (разделитель — запятая).
- `gender` — пол клиента (`0` — мужской, `1` — женский).
- `price_list_id` — ID прайс-листа по умолчанию (с версии 24.4).
- `birthday` — день рождения (формат `dd.mm.yyyy`).
- `date_last_ds` — дата применения последней ДС.

## BonusesBetweenForAll

Получение списка бонусов клиента за период. Метод: **POST** (большие периоды → используй `callback_address`).

```
POST .../api/?BonusesBetweenForAll
```

Тело (json):
```json
{
  "BonusesBetweenForAll": {
    "contr_id": "10013",
    "StartDate": "01.03.2021",
    "StopDate": "30.03.2021",
    "callback_address": "https://test.test/test.php"
  },
  "SessionID": "0BE170AB-86AF-441F-8523-8C960EA3EC4E"
}
```

Параметры:
- `contr_id` — ID клиента.
- `StartDate` — дата начала (**обязательный**).
- `StopDate` — дата окончания (**обязательный**).
- `callback_address` — https-адрес для результата (**необязательный**, с версии 24.4; см. общие соглашения).
- `SessionID` — сессия.

Ответ:
- `bonuses` — массив объектов **начислений** бонусов:
  - `bonus_id` — ID бонуса.
  - `parent_bonus_id` — ID родительского бонуса (в связке с `bonus_id`).
  - `doc_date` — дата начисления бонусов.
  - `doc_time` — время начисления бонусов.
  - `kredit` — сумма начисления бонуса.
  - `already_payed` — потрачено бонуса.
  - `bonus_rest` — остаток бонуса.
  - `active_before` — дата, до которой действуют бонусы.
  - `max_percent_in_zakaz` — максимальный процент оплаты бонусами.
  - `one_time` — признак «только разовое использование».
  - `comment` — комментарий.
  - `bonus_type_id` — ID типа бонусного начисления.
  - `bonus_type_name` — наименование типа бонусного начисления.
- `bonus_payments` — массив объектов **оплат** бонусами:
  - `id` — ID оплаты бонусом.
  - `bonus_id` — ID бонуса, которым оплачивали.
  - `doc_date` — дата оплаты бонусом.
  - `doc_time` — время оплаты бонусом.
  - `debet` — сумма оплаты бонусом.
  - `dor_id` — ID заказа.
  - `doc_num` — номер заказа.
