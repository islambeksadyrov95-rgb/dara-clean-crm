## Виды команд (5 категорий)
1. **Без сессии** (`no_session`) — общие справочники, авторизация. → `01-no-session.md`
2. **Только клиентская сессия** (`client_session`) — данные конкретного клиента (ЛК). → `02-client-session.md`
3. **Только пользовательская сессия** (`user_session`) — изменившиеся заказы/клиенты для синхронизации. → `03-user-session.md`
4. **Клиентская ИЛИ пользовательская** (`double_session`) — инфо по заказу, выезды, квитанции. → `04-double-session.md` (+ `04a-triporder.md`)
5. **Коммерческие (запись)** (`commercial_session`) — создание/изменение клиентов, заказов, оплат. **Тарифицируются.** → `05-commercial-session.md`

**Две сессии:**
- **Клиентская** — авторизация клиента (`ModernLogin`/`AuthByAddon`). Живёт неограниченно. Доступ только к данным этого клиента. Для синхронизации всех заказов **не подходит** (избыточная нагрузка).
- **Пользовательская** — авторизация сотрудника (`Login`, данные пользователя Агбис.Химчистка). Живёт **10 минут**, обновляется `RefreshSession`. Для синхронизации всех заказов/клиентов — использовать ЭТУ сессию + команды загрузки изменившихся.

## Формат команд
```
GET /himinfo.ru/cl/{Path}/api/?command={"key":"value"}&SessionID=...&callback=...
```
- Метод: `GET` или `POST`. Ответ — JSON.
- Все значения параметров — URL-encoded (`encodeURIComponent` / `urlencode`).
- `{Path}` — имя химчистки (у нас `daraclean_838936e8`).
- `SessionID` — для команд с сессией.
- `callback` — необязательный, имя процедуры для JSONP.
- ⚠ В ответах **строковые значения тоже URL-encoded** — декодировать на нашей стороне.
- В ответе могут быть поля вне документации — игнорировать.

## Авторизация
- Регистрация клиента: `ModernRegistration`. Авторизация клиента: `ModernLogin`, `AuthByAddon`. Восстановление: `ModernRememberPwd`.
- Авторизация пользователя (для коммерции/синхронизации): `Login` → `Session_id` (10 мин) + `Refresh_id`. Обновление: `RefreshSession`.

## Синхронизация изменившихся данных (для нашей CRM)
- **Изменившиеся заказы:** `OrderByDateTimeForAll` (подробно, за период) и `LastChangeOrder` (только ID + статус).
- **Изменившиеся клиенты:** `ClientsByDateTimeForAll`.
- Полная инфо по заказу: `FullOrderInfo`. Инфо по клиенту: `ContrInfo`.

## Работа с выездами
Журнал выездов — `Trips`. Резерв/редактирование — `TripOrder`. Перед резервом: `TripsHr` (свободное начало) → `TripsHrTo` (свободное окончание). После — `Trips` вернёт обновлённый список.

## Статистика ЛК (отчёт обращений)
`Entry` (0 регистрация/1 авторизация), `EntranceSite` (заходы), `OpenOrders`, `OpenHistory`.

## Пример работы с обсуждениями
`TitleMessages` (темы по сессии) → `MessageList` (сообщения темы по `id`) → `SendMessage` (в тему по `id` или новая без `id`; отзыв о заказе — с `dor_id`+`star`) → `CountNewMessages` (новые по `id`).

## Примеры интеграции (из доки)
PHP — `curl` + `urlencode('{"param1":"..."}')`. JS — `encodeURIComponent` + `XMLHttpRequest`. jQuery — `$.ajax` с `dataType:'jsonp'`, сам кодирует URL.

## Лимиты
- `ModernRegistration`, `ModernRememberPwd` — не более 3 запросов / 10 сек с IP.
- `Login` — сессия 10 мин; превышение → `429 Too many requests` на 2 мин.
- `TripOrder` по клиентской сессии — не более 5 выездов в день.
