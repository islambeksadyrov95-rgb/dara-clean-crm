### Креды и формат Login (gotchas)
- **User = `Дарын` (кириллица!)**, пользователь **программы** Агбис.Химчистка (User_ID 1022). НЕ латиница, НЕ аккаунт ХимИнфо.
- `Pwd` = SHA-1 пароля. Хранить в env (`AGBIS_API_USER`, `AGBIS_API_PWD`), **не в git**.
- **Кодировка:** значение в JSON — литералом (`"User":"Дарын"`), затем **весь** параметр `Login` URL-кодируется ОДИН раз (`encodeURIComponent` всей строки). Двойное кодирование значения → `error 1`.
- `ValidSessionID` — формат `?ValidSessionID&SessionID=<sid>` (не `=<sid>`).
- **Большие периоды (`OrderByDateTimeForAll`/`ClientsByDateTimeForAll`) → 504.** Для широких окон обязателен `callback_address` (асинхронный ответ POST'ом к нам). Узкие окна (дни/недели) тянутся синхронно.

## Глобальные правила API
- `GET`/`POST`, ответ — JSON. Все значения параметров **и строки в ответах** — URL-encoded.
- Авторизация чтения справочников **не нужна** (команды без сессии).
- Пользовательская сессия (`Login`) живёт **10 мин** → `RefreshSession`. Превышение → `429` на 2 мин.
- Тяжёлые POST-команды синхронизации поддерживают `callback_address` — результат приходит асинхронно POST'ом на наш URL (5 попыток при недоступности). Иначе большой период падает по timeout.
- Тарифицируются только записывающие команды (`*ForAll` записи + `TripOrder`): до 1000/мес — 3000 руб, сверх — 3 руб/запрос.

## Технические заметки для реализации
- Декодировать ВСЕ строковые значения ответа (`decodeURIComponent`).
- Деньги (`kredit`, `debet`, `price`) приходят строкой с десятичной **запятой** (`"801,93"`) — нормализовать перед парсингом; в CRM хранить в tenge/integer.
- Даты Agbis: `dd.mm.yyyy` и `dd.mm.yyyy HH:MM:SS`; в CRM — ISO/UTC.
- Для синхронизации использовать пользовательскую сессию + `callback_address` (не клиентскую — она не для массовой выгрузки).

## Архитектура подключения (как химчистка связана с himinfo)

Из внутренней инструкции MiniHim «подключение к ХимИнфо» (`doc.minihim.ru/ru/подключение_лк`, `1_алгоритм_подключения`):

```
Агбис.Химчистка (Firebird, on-premise сервер химчистки)
        │  ← локальная служба "Агент" (AgbisAgent / AgbisAgentReplService)
        │     связь: прямой IP:83  ИЛИ  SSH-туннель через im-gate.com
        ▼
Облако ХимИнфо (himinfo.ru/.org)  ← аккаунт заводится в программе «Управление ХимИнфо» (HimInfo.exe)
        ▼
API:  https://himinfo.org/cl/{Path}/api/   (тонкий шлюз, своих данных нет — проксирует в БД через Агента)
```

- Аккаунт тенанта (`daraclean_838936e8`) — запись в «Управление ХимИнфо» (логин латиницей без пробелов). **Заводит MiniHim вручную.**
- На каждый data-запрос шлюз идёт в живую БД химчистки **через соединение Агента**. Нет связи Агента → шлюз отвечает `108`.
- Точка входа реально `/api/index.php` (формы `/api/`, `/api`, `/api/index.php` — один код). Команды читаются **только из GET query string**. JSONP через `&callback=` поддерживается. DB-команды **троттлятся (429)** при частых повторах — повторы разносить.

## Troubleshooting `error 108` «БД химчистки недоступна»

Диагноз (доказан исчерпывающе, 15.06.2026):
- ✅ Работают только шлюзовые команды без обращения к БД: `api_version` (1.3), `ExecutedApiCount` (`error 0`) → **аккаунт на himinfo заведён, шлюз и маршрутизация исправны**.
- ⛔ `108` на ВСЕХ командах, которым нужна БД (включая `Login` — авторизация идёт против той же БД). Состояние **детерминированное**, не плавающее: 12/12 `PriceList` за 90с = 108; прогрев 8/8 не помогает.
- Не лечится ничем со стороны клиента: ни сменой credentials/хэша, ни заголовками, ни формой URL, ни POST→GET, ни JSONP. Альтернативных хостов нет (`agb.is` → 403, `lk.himinfo.ru` → нет DNS).

**Корень:** операционная БД Агбис.Химчистки тенанта `daraclean_838936e8` недоступна со стороны шлюза himinfo. Барьер на стороне провайдера/источника, не в нашем коде.

Что проверить, чтобы поднять связь:
- **Сторона Dara Clean (локальный админ Agbis):** сервер/ПК с БД включён и в сети; службы `Running`: `FirebirdServerDefaultInstance`, `AgbisAgentReplService`/`AgbisAgent`, `AgbisLicensingService`; в Агбис.Химчистка → Сервис → «Центр управления ХимИнфо» включена выгрузка; пользователь `daaaryn` существует и помечен для API.
- **Сторона MiniHim/Agbis (Леонид):** активен ли коммерческий API-модуль для пути; поднята ли облачная привязка БД для `daraclean_838936e8`; выполнить `Login` со своей стороны, наблюдая за сервером (доходит ли запрос до БД).
- Эскалация в Agbis при server-side проблеме: **+7 909 518-4444** (Пн–Пт 05:00–19:00 МСК), **support@agbis.ru**.

## Как только `108` исчезнет — проверочная цепочка (всё GET, повторы разносить)
1. `Login` → ждём `{"error":0,"Session_id":"…","Refresh_id":"…"}` (вместо 108). `error:2` = неверные credentials.
2. `ValidSessionID=<Session_id>` → подтвердить сессию (без значения даёт 103).
3. `PriceList={"price_id":"0"}` + сессия → реальный прайс вместо 108 = вся цепочка шлюз→Агент→Firebird поднята.
