# Dara Clean — Telegram Finance Bot

## Описание проекта
Telegram-бот для ведения финансового учёта компании Dara Clean (клининговая компания, Казахстан). Бот подключён к Google Sheets таблице «Финансы DaraClean 2025-2026» и позволяет сотрудникам записывать расходы/доходы, просматривать ДДС, баланс счетов и аналитику прямо в Telegram.

## Стек
- **Runtime**: Node.js + TypeScript (ESM)
- **Bot framework**: grammY + @grammyjs/conversations
- **Google Sheets**: google-spreadsheet + google-auth-library (сервисный аккаунт)
- **Хостинг**: Railway (long polling, не webhook)
- **Сборка**: `tsc` → `dist/`, запуск через `node dist/index.js`

## Важные правила

### Золотое правило
- **НИКОГДА не выдумывать данные** — все цифры только из таблицы. Если данных нет — писать "Нет данных", не 0.

### Категории доходов (ТОЛЬКО эти)
- Финансовые операции_1 → Получение банковского кредита, Получение займа
- Пополнение → Kaspi Pay, Наличными
- Услуги → Чистка ковров, Чистка штор, Химчистка мебели, Пледы и одеяла
- **ВСЁ остальное = Расход**

### Запись в таблицу
- **ВСЕГДА** писать в последнюю строку (после реальной последней строки с данными в колонке A)
- НЕ использовать `sheet.addRow()` — он вставляет после первой пустой строки, а не в конец
- Использовать `sheet.loadCells()` + `sheet.getCellByA1()` + `sheet.saveUpdatedCells()`
- **НЕ ставить `'` (апостроф) перед значениями** — Google Sheets интерпретирует его как text prefix
- Колонка G (Сотрудник) — всегда пустая
- valueInputOption при записи: `USER_ENTERED` (не `RAW`)

### Форматирование сообщений
- **НЕ использовать Markdown parse_mode** — символ `_` в названиях категорий (`Поставщик_1`) ломает парсинг
- Использовать эмодзи для визуального выделения вместо bold/italic
- Автоудаление предыдущих сообщений бота (кроме уведомлений)
- Loading-индикатор (`⏳ Загружаю...`) для всех тяжёлых операций

### Деплой
- Хостинг: Railway (проект "Dara Clean Bot")
- Деплой: `npx tsc && railway up`
- При деплое возможен 409 Conflict (старый/новый контейнер пересекаются) — Railway автоматически перезапустит
- Переменные окружения: `BOT_TOKEN`, `GOOGLE_SPREADSHEET_ID`, `GOOGLE_SERVICE_ACCOUNT_EMAIL`, `GOOGLE_PRIVATE_KEY`, `TZ=Asia/Almaty`

### Визард добавления операции
Порядок шагов (соответствует порядку столбцов в таблице):
1. Дата → сегодня или выбрать из календаря
2. Тип операции → Приход/Расход
3. Тип оплаты → из справочника
4. Сумма → ввод текстом
5. Категория → из справочника
6. Статья → из справочника (зависит от категории)
7. Комментарий → текст или "-" пропустить

### Календарь
- Drill-down навигация: дни → нажатие на месяц → выбор месяца → нажатие на год → выбор года (±5 лет)

### Вечернее напоминание
- Время: 20:30 Asia/Almaty
- Проверяет наличие записей за сегодня
- Если нет — отправляет мотивирующее сообщение всем пользователям
- Напоминания НЕ удаляются (остаются в истории как логи)
